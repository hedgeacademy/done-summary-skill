# Done Summary Checklist

Use this checklist before handing off a Done Summary.

- [ ] The goal is stated in one or two sentences.
- [ ] The summary says who did what.
- [ ] Key decisions and tradeoffs are explained.
- [ ] The final result is concrete and points to files, links, commits, artifacts, or outputs when relevant.
- [ ] Verification is specific: commands, checks, screenshots, builds, tests, or manual QA.
- [ ] Failed or skipped verification is stated clearly.
- [ ] Pitfalls include symptoms, causes, and fixes.
- [ ] Remaining risks and open items are not hidden.
- [ ] The next step is actionable, or the summary says no immediate next step is needed.
- [ ] Reusable knowledge is separated from one-time task details.
- [ ] The language matches the user's language unless the user requested otherwise.
