# Done Summary

## Goal

State the problem to solve or the result to deliver.

## Who Did What

Describe what the user, AI agent, tools, and external systems did.

## Why This Approach

Record the key decisions, tradeoffs, evidence, and assumptions. Mark inferred causes as inferred.

## Final Result

State what is complete, where the output lives, and how the user can use it now.

## Verification

List tests, builds, checks, screenshots, comparisons, manual QA, or production validation.  
If verification was not run, state why and what risk remains.

## Pitfalls

Record errors, symptoms, causes, and fixes. Focus on issues likely to recur.

## Risks And Open Items

State remaining uncertainty, edge cases, unverified areas, or follow-up risks.

## Next Step

State the most useful next action. If the work is closed, write “No immediate next step.”

## Reusable Knowledge

List what should be written to project docs, long-term memory, tool notes, deployment notes, or debugging runbooks.
