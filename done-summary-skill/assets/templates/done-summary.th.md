# สรุปงานเสร็จ

## เป้าหมาย

อธิบายปัญหาที่ต้องแก้ไข หรือผลลัพธ์ที่ต้องส่งมอบในครั้งนี้

## ใครทำอะไร

สรุปว่าผู้ใช้, AI agent, เครื่องมือ และระบบภายนอกแต่ละส่วนทำอะไรบ้าง

## ทำไมเลือกแนวทางนี้

บันทึกการตัดสินใจสำคัญ ทางเลือกที่พิจารณา เหตุผล หลักฐาน และสมมติฐาน หากเป็นสาเหตุที่อนุมาน ให้ระบุให้ชัดเจนว่าเป็นการอนุมาน

## ผลลัพธ์สุดท้าย

ระบุว่าส่วนใดเสร็จแล้ว ผลลัพธ์อยู่ที่ไหน และผู้ใช้สามารถนำไปใช้ต่อได้อย่างไร

## การตรวจสอบ

ระบุการทดสอบ การ build การตรวจสอบ screenshot การเปรียบเทียบ การ QA ด้วยมือ หรือการตรวจสอบบน production ที่ได้ทำแล้ว  
หากยังไม่ได้ตรวจสอบ ให้ระบุเหตุผลและความเสี่ยงที่ยังเหลืออยู่

## ปัญหาที่พบระหว่างทาง

บันทึก error อาการ สาเหตุ และวิธีแก้ไข โดยเน้นปัญหาที่อาจเกิดซ้ำได้ในอนาคต

## ความเสี่ยงและรายการที่ยังเปิดอยู่

ระบุความไม่แน่นอนที่ยังเหลืออยู่ edge cases พื้นที่ที่ยังไม่ได้ตรวจสอบ หรือความเสี่ยงที่ควรติดตามต่อ

## ขั้นตอนถัดไป

ระบุการดำเนินการถัดไปที่มีประโยชน์ที่สุด หากงานปิดสมบูรณ์แล้ว ให้เขียนว่า “ยังไม่มีขั้นตอนถัดไปที่จำเป็นทันที”

## ความรู้ที่ควรนำกลับมาใช้ซ้ำ

สรุปสิ่งที่ควรบันทึกไว้ในเอกสารโปรเจกต์ ความจำระยะยาว บันทึกเครื่องมือ คู่มือ deploy หรือ runbook สำหรับการแก้ปัญหา
